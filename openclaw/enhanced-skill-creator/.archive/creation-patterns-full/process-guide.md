# Process Guide 创建经验

## 核心模式

**决策点要显式**：
- 每个分支条件写清楚判断依据
- 用表格或决策树，不要用长段落
- "如果 X 则 A，否则 B" 比泛泛的"根据情况选择"有用得多

**步骤粒度要对**：
- 太粗 → agent 不知道具体做什么
- 太细 → 丧失灵活性，agent 变成打字机
- 每步应该是一个原子动作或一个明确判断

**渐进式**：
- 先给简版流程（Quick Reference），再给详细版
- 复杂流程拆分为子流程，各自独立可查
- 状态流转用 ASCII 图或表格表示

## 常见陷阱

- 写成流水账（Step 1, Step 2, Step 3...）但没有决策分支 → 不是 guide，是 checklist
- 跳过异常路径 → agent 遇到错误不知道怎么办
- 流程步骤有隐含依赖（必须先做 A 才能 B）但没写出来 → 顺序错乱
- 不提供跳回/回退机制 → 出错只能从头开始

## 质量信号

- 新手跟着走一遍能完成流程，不需要额外问人
- 异常路径有明确的回退策略
- 流程可以从中断点恢复
