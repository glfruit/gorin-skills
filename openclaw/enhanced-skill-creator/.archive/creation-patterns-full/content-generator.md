# Content Generator 创建经验

## 核心模式

**风格参数化**：
- 不在 prompt 里写死风格，而是用参数选择（风格 × 调色板 × 渲染方式）
- 提供默认值，但让用户可以覆盖
- 风格组合写在 references/ 或 assets/ 里，SKILL.md 只引用

**输出格式一致性**：
- 同类技能的输出格式保持一致（文件命名、目录结构）
- 给明确的输出路径规则
- 生成后验证文件存在且非空

**模板 + 变量 > 自由生成**：
- 内容生成类技能最怕"每次结果质量不稳定"
- 用模板确定结构，变量填内容
- 留有限的创造空间，不要全开放

## 多步骤生成

1. 先生成大纲/方案 → 用户确认 → 再生成内容
2. 长内容分步生成（分页、分段、分文件）
3. 每步生成后验证，失败可回退单步

## 常见陷阱

| 陷阱 | 表现 | 解决 |
|------|------|------|
| 风格不稳定 | 每次生成结果差异大 | 用固定 prompt 模板 + 参考图 |
| API 限流 | 批量生成到一半 429 | 控制并发 + 间隔 |
| 文件命名混乱 | 输出文件名不可预测 | 统一命名规则 `{type}-{date}-{title}` |
| 生成质量波动 | 有时好有时差 | 保存好的输出作为 few-shot 示例 |
| 资源浪费 | 生成后不需要 | 先确认再生成，或提供预览/dry-run |

## 验证清单

- [ ] 风格参数可配置
- [ ] 输出路径规则明确
- [ ] 有 dry-run 或预览模式
- [ ] 批量生成有并发控制
