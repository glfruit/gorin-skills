# 上海交通大学出版社 — 教材编写要求（默认）

> 来源：教师提供的《电子信息类教材编写通用要求》提取
> 日期：2026-04-08

## 一、体例与结构

| 项目 | 要求 |
|------|------|
| 体例格式 | 项目-任务式或章节式，全书保持统一 |
| 篇幅 | Word 180-240 页（五号字，1.2 倍行距） |
| 层级结构 | 结构统一、层级清晰（建议不超过四级标题） |

## 二、图片规范

- 图片清晰度必须达标（印刷要求 ≥300dpi）
- 图序、图题齐全（如"图1-1 CK6136数控车床外形图"）
- **禁用**：地图、领导人照片、公众人物照片
- 图片格式优先：矢量图（SVG/EPS）> 位图（PNG）> JPG
- 图片在 Word 中须居中，图题在图下方居中

## 三、公式规范

- 使用 MathType 录入，不得使用公式图片
- 公式编号格式：`(X-Y)`（X=章号，Y=公式序号）
- 公式居中，编号右对齐

## 四、代码规范

- 代码以文字形式呈现，不得使用代码截图
- 缩进统一使用 4 个空格
- 交稿前所有代码必须重新运行验证正确性
- 代码块标注编程语言

## 五、表格规范

- 表格以 Word 原生表格呈现，不得使用表格截图
- 表序、表题、表头齐全（如"表1-1 数控车床型号含义"）
- 表题在表上方居中
- 表格尽量不跨页；跨页时重复表头

## 六、中英文格式

- 专有名词格式：中文（英文全称，英文简写）
  - 示例：数控装置（Computer Numerical Control，CNC）
- 英文简写首次出现时必须给出全称
- 专有名词首字母大写
- 中英文之间留一个半角空格

## 七、AI 融入方式（三种）

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 拓展 AI 应用 | 在拓展知识板块介绍 AI 在本领域的应用 | 理解前沿趋势 |
| AIGC 辅助学习 | 用 AI 工具辅助学生理解/练习 | 课后练习、自主学习 |
| AI 工具替换 | 用 AI 工具替代传统工具/方法 | 技能训练 |

## 八、思政融入方式

- **素养目标**：在素质目标中融入职业精神、工匠精神
- **理论内容融入**：在知识讲解中自然穿插典型案例
- **实践案例融入**：在任务实施中使用中国制造案例
- 原则：自然融入，不生硬

## 九、辅文要求

| 辅文 | 要求 |
|------|------|
| 内容简介 | 300 字左右 |
| 前言 | 含教材特色说明、适用对象、编写思路 |
| 参考文献 | ≥ 5 条，含近两年文献 |
| 目录 | 自动生成，精确到三级标题 |

## 十、齐清定标准

| 标准 | 说明 |
|------|------|
| 齐 | 结构统一、层级清晰、格式一致 |
| 清 | 内容确定、无占位符、无歧义 |
| 定 | 经审阅确认，无待修改标记 |

## 十一、用语规范

- 计算机用语遵循国家标准 GB/T
- 中英文缩写格式全书统一
- 同一概念全书使用同一术语（如"数控车床"不混用"CNC车床"）
