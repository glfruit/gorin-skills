# ZK 笔记系统使用指南

> 统一笔记入口，单关键词 `zk` 触发

---

## 快速开始

**只记一个词：`zk`**

```
zk 刚和沙鸥讨论了教材体例设计，确定了工作过程系统化+贯穿项目的组合
```

系统自动判断为 **会议记录** → 保存到 Efforts/1-Projects/

---

## 触发方式

| 方式 | 示例 |
|------|------|
| **主触发** | `zk 内容...` |
| **Agent兼容** | `保存到笔记` / `save to notes` |

---

## 自动判断示例

| 你输入 | 系统判断 | 保存位置 |
|--------|---------|----------|
| `zk https://mp.weixin.qq.com/...` | 文献笔记(文章) | Zettels/2-Literature/Articles/ |
| `zk 突然想到一个产品功能...` | 想法笔记 | Zettels/1-Fleeting/ |
| `zk 和团队讨论了架构方案...` | 会议记录 | Efforts/1-Projects/ |
| `zk 下周计划完成第三章...` | 计划任务 | Efforts/1-Projects/ |
| `zk 读《深度工作》有感...` | 文献笔记(书籍) | Zettels/2-Literature/Books/ |
| `zk 本周工作总结...` | 周回顾 | Calendar/Reviews/ |

---

## 置信度与响应

### 高置信度 (≥80%)
✅ 直接保存，显示确认信息

### 中置信度 (50-80%)
✅ 保存 + 提示可调整类型

### 低置信度 (<50%)
🤔 询问确认类型

---

## 快速调整

保存后如需改类型，回复：

| 指令 | 改为 |
|------|------|
| `改zku` | 文献笔记(文章) |
| `改zkl` | 文献笔记(书籍) |
| `改zki` | 想法笔记 |
| `改zkm` | 会议记录 |
| `改zkt` | 计划任务 |
| `删除` | 删除刚保存的笔记 |

---

## 底层技能说明

`zk` 是统一入口，底层调用：

| 判断类型 | 调用技能 | 说明 |
|----------|----------|------|
| literature | zk-para-zettel | 文献摘录、Web保存 |
| idea | idea-creator | 灵感想法 |
| meeting/plan/review | pkm-save-note | 通用笔记 |

---

## 旧触发词兼容

以下触发词仍可用（底层不变）：

- `摘录` → 同 `zk` + URL
- `想法` → 同 `zki`
- `保存笔记` → 同 `zk`

但推荐使用统一的 `zk`。

---

## Agent 使用

任意Agent说"**保存到笔记**"自动触发 zk-router：

```
edu-tl: "这段讨论保存到笔记"
→ 自动判断为会议纪要
→ 保存到项目目录

daily-collector: "这篇好文章保存到笔记" (含URL)
→ 自动判断为文献笔记
→ 保存到 Literature/Articles/
```

---

## 系统架构

```
用户输入: zk <内容>
    ↓
zk-router (智能判断)
    ↓
┌─────────────┬─────────────┬─────────────┐
│ literature  │ idea        │ meeting/... │
↓             ↓             ↓
zk-para-    idea-       pkm-save-
zettel      creator     note
```
