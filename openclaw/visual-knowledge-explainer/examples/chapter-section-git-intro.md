# Module 1: Git 简介

## 核心概念

Git 是一个**分布式版本控制系统**（Distributed Version Control System, DVCS）。

### 什么是版本控制？

版本控制记录文件的每次修改，让你可以：
- 回到之前的版本
- 查看修改历史
- 协作开发而不互相覆盖

### 什么是分布式？

每个开发者都有完整的代码库副本，包括所有历史记录。

---

## 常见误区

### 误区 1: Git 只用于代码

**错误**: "Git 只能管理代码"

**事实**: Git 可以管理任何文本文件：
- 文档（Markdown、HTML、LaTeX）
- 配置文件
- 图片（二进制文件）
- 数据文件

---

## 应用场景

### 个人项目
- 保护你的代码
- 测试新功能而不影响主分支
- 一键回滚到稳定版本

### 团队协作
- 多人同时开发
- 代码审查（Pull Request）
- 问题追踪

---

## 小结

- Git 是分布式版本控制系统
- 所有修改都有记录
- 每个开发者有完整副本
