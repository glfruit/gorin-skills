# edu-outline-review

## Core Principles
- 对照 Excel 大纲逐行核对，不做主观教学质量判断。
- 输出可执行、可定位的问题列表，优先给出缺失项、顺序问题、不一致项。
- 审查重点是“大纲符合度”，不是“内容写得好不好”。

## When to Use
- 手稿 Markdown 已经完成，需要进入 Phase 2 第一阶段审查时。
- 需要确认手稿是否覆盖 Excel 大纲中的模块、知识点、任务目标时。
- ❌ 如果要评审教学设计质量、表达质量、教学活动有效性，请改用 `edu-critic`。

## Workflow
1. 读取 Excel 大纲。
2. 提取手稿 Markdown 的二级、三级标题与正文结构。
3. 按模块/板块逐项映射知识点。
4. 输出 Markdown 审查报告。

## 检查维度
1. 板块覆盖率：大纲中的每个模块/板块在手稿中是否存在。
2. 知识点覆盖率：大纲中每个知识点在手稿中是否有对应内容。
3. 结构顺序：手稿板块顺序是否与大纲一致。
4. 教学目标动词合规：检查禁用虚词（了解、掌握、熟悉、培养、提高）。
5. 占位符扫描：TBD、待定、TODO。

## 输出格式
- 输出 Markdown 报告。
- 必须包含覆盖率百分比、缺失项列表（含大纲行号）、结构顺序问题、不一致项。
- 结论统一给出 pass/fail 风格总评。

## Gotchas
- 教材大纲前几行可能是课程元信息，第 5 行才是表头，读取时不要假设永远从第 1 行开始。
- 手稿板块名可能与大纲有细微差异，例如“相关知识”与“相关知识（理论）”，需要模糊匹配。
- 同一项目/任务下，Excel 里可能出现合并单元格，解析时要向下继承上方值。
